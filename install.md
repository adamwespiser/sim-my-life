# Install

This app is shipped as a single JavaScript bundle plus one custom element tag.

The production artifact is:

- `dist/sim-returns.js`

The embed contract is:

```html
<sim-returns-app></sim-returns-app>
```

## What You Need

- a web page on your site
- the built bundle copied somewhere your site can serve over HTTP
- a `<script type="module">` tag that loads the bundle

Do not open the page with `file://`. Load it from your website or a local HTTP server.

## Build The Bundle

From this repo:

```bash
npm install
npm run build
```

That produces:

- `dist/sim-returns.js`

## Simplest Install

1. Copy `dist/sim-returns.js` into your website's static assets.
2. Add a page that loads that file with a module script.
3. Place `<sim-returns-app></sim-returns-app>` where you want the app to appear.

Example:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Sim Returns</title>
    <script type="module" src="/assets/sim-returns.js"></script>
  </head>
  <body>
    <main>
      <sim-returns-app></sim-returns-app>
    </main>
  </body>
</html>
```

## If Your Site Has A Static Assets Folder

Typical mapping:

- source repo output: `dist/sim-returns.js`
- website asset path: `public/assets/sim-returns.js` or similar

Then reference it like:

```html
<script type="module" src="/assets/sim-returns.js"></script>
<sim-returns-app></sim-returns-app>
```

## If Your Site Is Generated By Another App

If your main website has its own build system, the easiest path is usually:

1. build this repo separately
2. copy `dist/sim-returns.js` into the website's public/static directory
3. reference that copied file from the page or template where you want the app

You do not need to import this app into React, Vue, or another framework runtime. It is already a custom element.

## Optional Page Wrapper

You can wrap it in your own container:

```html
<main class="page-shell">
  <h1>Sim Returns</h1>
  <p>Try different retirement scenarios.</p>
  <sim-returns-app></sim-returns-app>
</main>
```

The app renders inside Shadow DOM, so your site CSS will not directly style its internals.


```bash
npm run build
python3 -m http.server 4173
```

Then open:

- `http://localhost:4173/demo.html`

## Troubleshooting

### The app does not appear

Check:

- the script path is correct
- the script is loaded with `type="module"`
- the page is being served over HTTP
- the page contains `<sim-returns-app></sim-returns-app>`

### The app shows as plain text or an empty tag

That usually means the JavaScript bundle did not load, or loaded from the wrong path.

### The page loads locally but not on the site

Check whether your site pipeline is actually publishing `sim-returns.js` to the final asset URL you referenced.
